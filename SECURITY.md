# Security policy

This project is intended for authorized AnyDesk maintenance and troubleshooting tasks only.

These scripts must not be used to bypass licensing, access control, auditing, usage restrictions or traceability mechanisms.

## Supported versions

| Version | Supported |
|---|---|
| Latest release | Yes |
| Older releases | No |

## Responsible use

Only run these scripts on systems where you have explicit administrative permission.

Before running any script, review its content and confirm that it matches your environment.

Do not execute remote scripts blindly from the internet.

Do not use this project to alter, hide, reset or regenerate licensing data, access controls or audit information.

## Reporting a security issue

If you find a security issue, do not publish it publicly.

Report it privately to the repository maintainer.

When reporting an issue, include:

- Operating system and version.
- Script name and version.
- Steps to reproduce the issue.
- Relevant log lines.
- Expected behavior.
- Actual behavior.

Do not include passwords, tokens, AnyDesk IDs, customer data, personal data, private connection details or organization-specific secrets.

## In scope

The following issues are in scope:

- Unsafe file deletion.
- Incorrect permission handling.
- Commands affecting unrelated applications.
- Logs exposing sensitive local information.
- Backup or restore behavior that may cause data loss.
- Script behavior that leaves AnyDesk in an unstable state.

## Out of scope

The following requests are out of scope:

- Bypassing AnyDesk licensing.
- Removing commercial-use restrictions.
- Hiding usage history.
- Disabling auditing or traceability.
- Unlocking paid features.
- Circumventing access controls.

## Disclosure expectations

Please allow reasonable time for review before sharing technical details publicly.

If the issue is valid, it will be reviewed and addressed in a future update.

Reports requesting bypasses, cracks, or license resets will be closed without action.
